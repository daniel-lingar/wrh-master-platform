# WRH Glossary

## Key Program Terms

This glossary defines recurring terms used throughout the **What Really Happened (WRH)** pilot package. The definitions are written to preserve the program's operational language and to help facilitators maintain consistency across sessions, documentation, and participant instruction.

---

| Term | Definition |
| :--- | :--- |
| **Pattern Veto** | A deliberate interruption move used when a participant detects the old sequence starting to take control. It is the act of recognizing the mechanism in real time and refusing to let the inherited pattern run the next step without challenge. |
| **The Interrupt** | The phase of the curriculum focused on breaking automatic survival routines and installing replacement actions. It represents the moment when the driver notices the system taking over and intervenes before the old code completes the loop. |
| **Micro-Regulation** | A small, repeatable regulation action used to lower system load without requiring a major reset. Examples include changing posture, pressing feet into the floor, lengthening the exhale, or naming the current mechanism before it accelerates. |
| **The System** | The full body-mind operating structure that carries old survival instructions, stress responses, and automatic behavior under pressure. In WRH, the system is treated as a mechanism that can be observed, mapped, and retrained. |
| **The Mechanism** | The specific way a pattern works from trigger to response to consequence. When facilitators explain the mechanism, they show how the sequence runs rather than judging the participant for the output. |
| **The Driver** | The conscious decision-making layer that can learn to observe the system, call out the pattern, and choose a different action. The driver is not always in control, but the curriculum is built to strengthen that control window. |
| **The Hum** | The low-grade internal alarm state that keeps the system scanning for pressure even when no immediate threat is visible. It functions like background engine vibration that never fully powers down. |
| **Threshold** | The point at which accumulated pressure exceeds the system's handling capacity and triggers automatic survival output. Once the threshold is crossed, reaction speed increases and choice narrows. |
| **The Loop** | A recurring pattern sequence in which trigger, reaction, consequence, and reinforcement keep feeding one another. WRH trains participants to identify loops early enough to interrupt them. |
| **Freeze** | A survival response in which the system prioritizes shutdown, immobility, or disappearance over confrontation or escape. WRH frames freeze as old code executing under load rather than as weakness or refusal. |
| **Dissociation** | A high-load protection response in which the system disconnects awareness from the immediate environment to reduce overload. The body remains present, but the driver's access to the moment is reduced or lost. |
| **Time Collapse** | A state in which the system reacts to present conditions as though an older threat is happening again right now. The mechanism does not register the event as memory; it treats it as current reality. |
| **Container** | The set of boundaries, pacing rules, and safety controls that keep the room stable enough for learning. The container protects instruction from drifting into chaos, disclosure pressure, or uncontrolled escalation. |
| **Ledger** | The internal record of what the system believes has worked before under pressure. If disappearance, shutdown, or self-protection once reduced danger, the ledger stores that result as preferred code. |

---

Facilitators should use these definitions consistently so the program retains a unified instructional voice across all sessions and support materials.
