# WRH Pilot Evaluation Framework (Form 01-A)
**Classification:** Pre-Registered Pilot Measurement Framework  
**Program Administrator:** Capitol Contracts LLC  

---

## 📌 Framework Purpose
This framework defines how a WRH pilot will be evaluated before the pilot begins. It prevents outcome cherry-picking, protects the credibility of pilot findings, and keeps Phase 1 evaluation inside the correct scope.

Phase 1 does **not** evaluate clinical effectiveness. It evaluates whether WRH can be delivered safely, consistently, and within non-clinical boundaries at one approved site.

---

## 🛡️ Scope Boundary
WRH is a non-clinical psychoeducational curriculum. The pilot evaluation measures operational performance, delivery fidelity, participant engagement, safety documentation, and support pathway function.

This framework does **not** measure, diagnose, or claim:

*   PTSD or complex PTSD symptom reduction
*   Suicide prevention outcomes
*   Addiction treatment outcomes
*   Clinical stabilization
*   Therapeutic effectiveness
*   Replacement of licensed clinical care

---

## 🎯 Phase 1 Evaluation Question
The central Phase 1 question is:

> **Can WRH be run safely once, measured honestly, and improved before it scales?**

The pilot answers this by testing:

*   Was the curriculum delivered as designed?
*   Did facilitators stay inside non-clinical scope?
*   Did participants engage without required trauma disclosure?
*   Were activation events identified and documented?
*   Did the Closed-Loop Support Bridge function when needed?
*   Were unresolved handoffs identified instead of hidden?
*   Did the Site Readiness Gate accurately predict site performance?
*   What must change before continuation or expansion?

---

## 🧱 Evaluation Domains

### 1. Implementation Domain
Measures whether the pilot was actually delivered.

*   Attendance rate
*   Completion rate
*   Sessions delivered as scheduled
*   Participant feedback form completion
*   Administrative form completion
*   Site coordinator responsiveness

### 2. Fidelity Domain
Measures whether facilitators delivered WRH as written and stayed within scope.

*   Facilitator Fidelity Scoring Sheet results
*   Non-disclosure rule adherence
*   Script / curriculum adherence
*   Boundary drift incidents
*   Stop Rule use when required
*   Critical safety item failures

### 3. Safety & Activation Domain
Measures whether participant distress was recognized, slowed, documented, and routed through the correct site pathway.

*   Activation Event Logs
*   Yellow / Orange / Red event counts
*   Support Bridge Activation Logs
*   Participant refusals
*   Participant departures while activated
*   Emergency / crisis protocol activations

### 4. Support Bridge Domain
Measures whether support connections were confirmed instead of merely referred.

*   Total support bridge activations
*   Completed support connections
*   Unresolved handoffs
*   Time from activation to outcome
*   Unresolved Handoff Review completion
*   Corrective actions required

### 5. Site Readiness Accuracy Domain
Measures whether the Site Readiness Gate accurately predicted whether the site could support WRH deployment.

*   Support lead reachability
*   Backup support lead reachability
*   After-hours procedure function
*   Documentation chain function
*   Escalation pathway clarity
*   Site remediation requirements

## 📊 Pre-Registered Metrics

| Metric | Source Form | Target / Review Standard |
| :--- | :--- | :--- |
| Attendance Rate | Session Attendance & Completion Tracker | Recorded each session |
| Completion Rate | Session Attendance & Completion Tracker | Recorded each session and final pilot |
| Facilitator Fidelity Rate | Facilitator Fidelity Scoring Sheet | Target ≥ 95% |
| Administrative Compliance Rate | Field form submission timestamps | Target ≥ 98% within required documentation window |
| Activation Events | Activation Event Log | Count and response level |
| Support Bridge Completion Rate | Support Bridge Activation Log | Confirmed connection or documented unresolved status |
| Unresolved Handoffs | Unresolved Handoff Review Form | Must be reviewed and escalated |
| Site Performance Score | Site Performance Summary | Green / Amber / Red decision |
| Required Revisions | Final Pilot Report + Revision Memo | Must be documented before expansion |

---

## 🧮 Site Performance Scoring Model
The Site Performance Summary uses the following weighted model:

$$S_p = 0.40(\bar{F}) + 0.30(B) + 0.15(A) + 0.15(\bar{E})$$

Where:

*   **$S_p$** = Final Site Performance Score
*   **$\bar{F}$** = Average Facilitator Fidelity Rate
*   **$B$** = Verified Closed-Loop Support Bridge Completion Rate
*   **$A$** = Administrative Compliance Rate
*   **$\bar{E}$** = Average Environment Safety Rating

### Authorization Thresholds
*   **GREEN LIGHT (Score ≥ 90):** Site met administrative, fidelity, and support pathway standards.
*   **AMBER LIGHT (Score 75–89):** Structural drift or administrative backlog requires revision decision before continuation.
*   **RED LIGHT (Score < 75 or $B = 0$):** Safety failure or severe boundary drift. Immediate suspension of program delivery at this site.

---

## 🛑 Critical Failure Rules
The following findings trigger automatic escalation regardless of total score:

*   Any critical facilitator boundary breach
*   Any unresolved handoff that is not reviewed and escalated
*   Any support bridge activation with no documented final status
*   Any site unable to reach its designated support lead and backup during an activation event
*   Any participant departure while activated with no follow-up procedure triggered
*   Any evidence that facilitators solicited trauma disclosure or drifted into clinical counseling

---

## 📝 Required Data Sources
The pilot evaluation must use only documented pilot records, including:

*   Participant Feedback Forms
*   Facilitator Fidelity Scoring Sheets
*   Session Attendance & Completion Trackers
*   Activation Event Logs
*   Support Bridge Activation Logs
*   Unresolved Handoff Review Forms
*   Site Performance Summary
*   Final Pilot Report Template
*   Pilot Revision Decision Memo

Undocumented impressions, informal comments, or unsupported clinical interpretations are not evaluation evidence.

---

## ✅ Claims WRH May Make From Phase 1 Data
If supported by the record, WRH may state:

*   WRH was delivered with fidelity at the pilot site.
*   Participants engaged without required trauma disclosure.
*   Facilitators stayed inside non-clinical scope.
*   Activation events were identified and documented.
*   The Closed-Loop Support Bridge functioned as designed.
*   Unresolved handoffs were identified, reviewed, and escalated.
*   The Site Readiness Gate accurately predicted site performance.
*   The pilot identified required revisions before expansion.

---

## 🚫 Claims WRH Will Not Make From Phase 1 Data
WRH will not claim:

*   WRH treats PTSD or complex PTSD.
*   WRH prevents suicide.
*   WRH reduces clinical symptoms.
*   WRH treats addiction.
*   WRH replaces therapy or licensed care.
*   WRH produces clinical outcomes.
*   WRH proves clinical effectiveness.
*   WRH facilitator training creates clinical competency.

---

## 🔁 Revision Requirement
A pilot is not a performance. A pilot is a stress test.

If the pilot identifies drift, safety gaps, administrative burden, unresolved handoffs, or support pathway failures, WRH must revise the relevant system layer before expansion.

Possible revision targets include:

*   Curriculum pacing
*   Facilitator scripts
*   Facilitator training
*   Participant orientation language
*   Site readiness gate
*   Support bridge procedure
*   Field forms
*   Documentation timelines
*   Pilot evaluation metrics
